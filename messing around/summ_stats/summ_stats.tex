\section{Intro}\label{intro}}

This doc has the code and results for each variable outlined
\href{https://docs.google.com/spreadsheets/d/1c49ib2oDodITWm66JfoQ4nshZbZ-YrHiZI8S4D1esHE/edit\#gid=0}{here}.
Results are being stored as CSVs for now (presumably we'll have a
different file management system in place; this is just a placeholder).
In addition, graphs/summary statistics are included for each variable.

Each section is dedicated to a specific variable. I include the SQL
query used, so it's easy to see if/where I'm making mistakes. \emph{All
queries are currently limited to January, 2015} (this is easily
expandable, just limiting them for now). Details regarding my connection
to Google BigQuery can be seen below.

\begin{verbatim}
## <BigQueryConnection>
##   Dataset: ghtorrent-bq.ght
##   Billing: mcd-lab
\end{verbatim}

\hypertarget{variable-1-number-of-distinct-teams-working-on}{%
\section{Variable 1: Number of distinct teams working
on}\label{variable-1-number-of-distinct-teams-working-on}}

Our first variable is the ``number of distinct teams working on'',
observed at the person-week level, to measure being on multiple teams
simultaneously. The query itself can be seen below.

\begin{verbatim}
## <SQL> SELECT author_id, EXTRACT(WEEK FROM date(created_at)) as week, COUNT(DISTINCT project_id) as num_projects  
## FROM `ghtorrent-bq.ght.commits`
## WHERE EXTRACT(YEAR FROM date(created_at)) = 2015 AND EXTRACT(MONTH FROM date(created_at)) = 1
## group by author_id, EXTRACT(WEEK FROM date(created_at))
\end{verbatim}
